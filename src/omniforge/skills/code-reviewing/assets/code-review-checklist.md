# Code Review Workflow Checklist

## Status Tracking
- [ ] Review Completed: ____/____/____
- [ ] Reviewer: _____________________
- [ ] Reviewed Version: _______________

## Code Review Steps

### 1. General Assessment
- [ ] Verify code compiles without errors
- [ ] Check for any obvious syntax errors
- [ ] Review overall code structure and organization

### 2. Functionality
- [ ] Verify all features work as intended
- [ ] Test edge cases and error handling
- [ ] Check for any potential bugs or issues

### 3. Code Quality
- [ ] Review code readability and maintainability
- [ ] Check for proper naming conventions
- [ ] Verify appropriate comments and documentation
- [ ] Ensure code follows project's style guidelines

### 4. Performance
- [ ] Check for any potential performance bottlenecks
- [ ] Review database queries for efficiency
- [ ] Verify appropriate caching mechanisms are in place

### 5. Security
- [ ] Check for potential security vulnerabilities
- [ ] Review input validation and sanitization
- [ ] Verify proper authentication and authorization

### 6. Testing
- [ ] Verify unit tests are present and passing
- [ ] Check integration tests for critical functionality
- [ ] Review test coverage and identify gaps

### 7. Dependencies
- [ ] Check for any outdated or vulnerable dependencies
- [ ] Verify proper license compliance for third-party libraries
- [ ] Review dependency management and version pinning

### 8. Documentation
- [ ] Verify API documentation is up-to-date
- [ ] Check for any required README updates
- [ ] Review inline code comments for clarity

### 9. Deployment Considerations
- [ ] Check for any deployment-specific configurations
- [ ] Verify environment-specific settings are properly handled
- [ ] Review database migrations or schema changes

### 10. Final Review
- [ ] Ensure all TODOs and FIXMEs are addressed
- [ ] Verify code adheres to project's coding standards
- [ ] Check for any unnecessary code or commented-out sections
